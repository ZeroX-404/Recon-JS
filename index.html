// CLI Reference
Perintah CLI yang sering dipakai bersamaan tool ini
1. Recon — Extract JS Files
$ getJS --url https://target.com --output js-urls.txt
$ katana -u https://target.com -jc -o endpoints.txt # js crawl
$ gospider -s https://target.com -t 10 --js | grep "\.js"
2. Download JS Contents
$ cat js-urls.txt | xargs -I{} curl -s {} -o {}.js
$ while read url; do curl -s "$url" >> all.js; done < js-urls.txt
3. Source Map Extraction
$ sourcemapper -url https://target.com/main.abc.js -output out/
$ python3 sourcemapper.py --url https://target.com/app.js.map
4. Secret Scanning CLI
$ trufflehog filesystem ./out/ --only-verified
$ gitleaks detect --source ./out/ --report-format json
$ grep -rE "(AKIA|ASIA)[A-Z0-9]{16}" ./out/
$ grep -rE "userPoolId|identityPoolId|clientId" ./out/
5. Endpoint Extraction CLI
$ linkfinder -i https://target.com/main.js -o cli
$ python3 LinkFinder.py -i ./all.js -o results.html
$ grep -oE '"(/[a-zA-Z0-9_/-]+)"' all.js | sort -u
6. Source Map Hunting dengan ffuf
$ ffuf -u https://target.com/FUZZ -w sourcemap-candidates.txt -mc 200
$ curl -s https://target.com/main.abc.js.map | python3 -m json.tool
// Workflow Guide
Recon methodology untuk JS files dalam bug bounty
1
Subdomain & JS Discovery
  • Gunakan subfinder / amass untuk subdomain
  • Fokus ke subdomain dev/staging yang sering kurang aman
  • Gunakan getJS / katana untuk extract semua JS URLs
  • Cek Network tab di DevTools saat load halaman
2
Source Map Unpacking
  • Cek // sourceMappingURL= di akhir JS file
  • Coba akses main.js.map langsung
  • Gunakan sourcemapper untuk extract source tree
  • Source maps sering berisi kode asli dengan komentar dan config
3
Secret Detection
  • Paste ke JS Analyzer tab ini
  • Prioritaskan: AWS keys, Cognito config, API keys
  • Gunakan Entropy Scanner untuk secrets tak dikenal
  • Double-check dengan trufflehog / gitleaks
4
Endpoint Mapping
  • Paste ke Endpoint Mapper
  • Filter endpoints yang menarik: /admin, /internal, /api
  • Uji endpoint yang ditemukan dengan Burp Suite
  • Cari hidden endpoints yang tidak terekspos di UI
5
What to Report
  • Exposed credentials / API keys → Critical/High
  • Cognito config yang memungkinkan self-registration
  • Source maps exposed di production
  • Internal API endpoints terekspos
  • Environment variables bocor (DEV_MODE, DEBUG)
6
Etika Bug Bounty
  • Selalu verifikasi scope sebelum test
  • Jika temukan credentials → jangan eksploitasi lebih jauh
  • Dokumentasikan proof of existence, bukan proof of exploitation
  • Report segera, jangan hold untuk eskalasi
